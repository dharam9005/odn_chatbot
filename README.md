## 🚀 Quick Start

### Prerequisites

- Python 3.9 or higher
- A free Groq API key → [console.groq.com/keys](https://console.groq.com/keys)

### 1. Clone the Repository
```bash
git clone https://github.com/yourusername/odn-chatbot.git
cd odn-chatbot
```

### 2. Create a Virtual Environment
```bash
# Create
python -m venv .venv

# Activate — macOS/Linux
source .venv/bin/activate

# Activate — Windows
.venv\Scripts\activate
```

### 3. Install Dependencies
```bash
pip install -r requirements.txt
```

### 4. Set Up Your API Key
```bash
# Copy the example file
cp .env.example .env
```

Open `.env` and add your Groq key:
```env
GROQ_API_KEY=gsk_your_actual_key_here
```

> 🔑 Get a free key at [console.groq.com/keys](https://console.groq.com/keys) — no credit card needed.

### 5. Build the Knowledge Base
```bash
python scraper.py
```

This crawls all pages of odnsystems.com and saves the content to `knowledge_base.json`.
```

    ✓ Services (2800 chars)
  ...
✅ Saved to knowledge_base.json
Pages scraped: 20
```

### 6. Start the Chatbot
```bash
python app.py
```

## 🧩 Key Features Explained

### Smart Context Retrieval
Instead of sending the entire 42,000-character knowledge base with every request, the chatbot scores each KB page by keyword overlap with the question and sends only the top 3 most relevant pages (~3,500 chars). This cuts token usage by **~75%** and speeds up responses.

### Automatic Retry Logic
If the Groq API returns a rate limit error, the app waits and retries automatically up to 3 times — the user just sees the typing indicator and never knows there was a hiccup.

### Session Memory
The last 6 turns of conversation are sent with every request, so the AI remembers context within a session (e.g., "tell me more about that").

### Branded UI
The interface uses ODN Systems' exact color scheme (`#0b2d6e` navy, `#1550b8` blue) with:
- Quick-topic chips for common questions
- Contextual follow-up suggestions after each response
- Smooth message animations and typing indicator
- Fully responsive for mobile

## 🔧 Configuration

All configuration is in `app.py`:
```python
MODEL             = "llama-3.3-70b-versatile"  # Change AI model here
MAX_SNIPPET_CHARS = 3500     # KB content per request (larger = smarter but slower)
MAX_RETRIES       = 3        # How many times to retry on rate limit
RETRY_DELAY       = 5        # Seconds between retries
```

### Available Groq Models (all free)

| Model | Speed | Best For |
|-------|-------|---------|
| `llama-3.3-70b-versatile` | Fast | **Best quality — recommended** |
| `llama-3.1-8b-instant` | Very fast | High-traffic / low latency |
| `mixtral-8x7b-32768` | Fast | Long context conversations |
| `gemma2-9b-it` | Fast | Lightweight alternative |

## 🌐 Embedding on the ODN Systems Website

To add the chatbot as a floating widget on the real website, paste this before `</body>` on any page:
```html
<script>
(function() {
  const btn = document.createElement('button');
  btn.innerHTML = '💬';
  btn.style.cssText = `
    position:fixed; bottom:28px; right:28px; width:58px; height:58px;
    border-radius:50%; background:linear-gradient(135deg,#1550b8,#1a6ef5);
    border:none; cursor:pointer; font-size:24px; z-index:9999;
    box-shadow:0 4px 20px rgba(21,80,184,.5); transition:transform .2s;
  `;
  btn.onmouseenter = () => btn.style.transform = 'scale(1.1)';
  btn.onmouseleave = () => btn.style.transform = 'scale(1)';

  const frame = document.createElement('iframe');
  frame.src = 'http://127.0.0.1:5000';  // ← change to your deployed URL
  frame.style.cssText = `
    position:fixed; bottom:100px; right:24px;
    width:420px; height:600px; border:none; border-radius:18px;
    box-shadow:0 20px 60px rgba(0,0,0,.35); z-index:9998; display:none;
  `;

  let open = false;
  btn.onclick = () => {
    open = !open;
    frame.style.display = open ? 'block' : 'none';
    btn.innerHTML = open ? '✕' : '💬';
  };

  document.body.appendChild(frame);
  document.body.appendChild(btn);
})();
</script>
```

## 🚀 Deploy to Production

### Railway (Recommended — Free Tier)

1. Push this repo to GitHub
2. Go to [railway.app](https://railway.app) → **New Project** → **Deploy from GitHub**
3. Select your repo
4. Go to **Variables** → Add `GROQ_API_KEY=gsk_your_key`
5. Deploy — you'll get a URL like `https://odn-chatbot.up.railway.app`

### Render (Alternative — Free Tier)

1. Go to [render.com](https://render.com) → **New Web Service**
2. Connect your GitHub repo
3. Set **Build Command**: `pip install -r requirements.txt`
4. Set **Start Command**: `gunicorn app:app`
5. Add environment variable `GROQ_API_KEY`
```bash
# Install gunicorn for production
pip install gunicorn

# Run in production mode
gunicorn -w 4 -b 0.0.0.0:5000 app:app
```

## ❓ Troubleshooting

| Error | Cause | Fix |
|-------|-------|-----|
| `❌ GROQ API KEY NOT SET` | `.env` file missing or wrong | Create `.env` with your key from console.groq.com |
| `ModuleNotFoundError: groq` | Dependencies not installed | Run `pip install -r requirements.txt` |
| `Port 5000 already in use` | macOS AirPlay Receiver | System Settings → General → AirDrop & Handoff → disable AirPlay Receiver |
| `knowledge_base.json not found` | Scraper not run yet | Run `python scraper.py` first |
| `429 RESOURCE_EXHAUSTED` | Rate limit hit | Wait 60 seconds — retry logic handles this automatically |
| `403 on localhost:5000` | macOS routing issue | Use `http://127.0.0.1:5000` instead of `localhost:5000` |
| Bot gives outdated answers | Website has changed | Re-run `python scraper.py` and restart `app.py` |
